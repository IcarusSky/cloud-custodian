1.1确保所有 CES 告警关联 SMN 主题
解释：通过检查AlarmActions字段，自动为未关联 SMN 主题的告警规则添加默认通知主题，确保所有告警事件都能通过 SMN 及时通知到运维团队。

yaml
policies:
- name: ces-alarms-have-smn
  description: "验证所有告警规则已关联SMN通知"
  resource: huaweicloud.ces.alarm
  filters:
    - type: value
      key: notification_enabled
      value: false
  actions:
    - type: update-alarm
      parameters:
        alarm_actions: ["urn:smn:cn-north-4:e196f2790965422f80502748f4d58649:CES_notification_group_kNrnzmm0J"]

policies:
  - name: enable-smn-notification
	resource: huaweicloud.alarm
	flters:
	  - type: value
		key: alarm_action_enabled
		value: false
	actions:
	  - type: ces-alarm-have-smn
		parameters:
		  action_type: "notification"
		  notification_list:
			- "urn:smn:cn-north-4:xxxxxxxxxxx:ces_notification_group_xxxxx"

1.2清理未使用的 CES 告警
解释：识别 30 天内未触发且状态为 OK 的冗余告警规则，自动删除这些无效配置，降低监控维护成本并提高规则有效性。
yaml
policies:
- name: ces-cleanup-stale-alarms
  description: "删除30天未触发的告警规则"
  resource: huaweicloud.ces.alarm
  filters:
    - type: age
      days: 30
      key: StateChangeTime
    - type: value
      key: StateValue
      value: OK
  actions:
    - type: delete

policies:
- name: ensure-alarm-enabled-check

  filters:
    - type: value
      key: alarm_enabled
      value: false
      op: eq
  actions:
    - type: notify
      to:
        - security@example.com
      subject: "Unverified CES Alarm"
      message: "CES alarm {alarm_id} is not started. Please verify the configuration."


1.4 alarm-kms-disable-or-delete-key
解释：CES必须配置监控KMS禁用或计划删除密钥的事件监控告警，若未配置，则标记为“不合规”
policies:
- name: alarm-kms-disable-or-delete-key
  resource: huaweicloud.ces.alarm
  filters:
    - type: value
      key: namespace
      value: SYS.KMS
      op: eq
    - type: value
      key: policies.metric_name
      value: scheduleKeyDeletion
      op: eq
    - type: value
      key: policies.metric_name
      value: disableKey
      op: eq
  actions:
    - type: notify
      to:
        - security@example.com
      subject: "Unverified CES Alarm"
      message: "The rule is noncompliant if an CES alarm for disabling or scheduled deletion of KMS keys does not exist. Please verify the configuration."
